4. 後端 API（Apps Script Web App）

呼叫參數行為
GET ?action=contract&token=token回合約全部欄位＋條文版本＋設備單價。token 無效或已簽 → 回 {state:"signed"} 供前端切唯讀
POST action=signtoken、equip、簽名 PNG（base64）驗 token → 寫 signed_* → 存簽名圖 → 產 PDF → status 改「在住」→ 回 PDF 連結
GET ?action=handover&token=token同上，回點交單內容
POST action=handoverSigntoken、七項結果、need_cleaning、簽名算賠償總額 → 產點交 PDF → 合約 status 改「已退宿」
POST action=create（管理端）合約欄位＋管理密碼建單、發 token、回連結。床位重複時回警告，帶 force=1 可強制建立
GET ?action=list（管理端）管理密碼回全部合約摘要供後台表格

管理端用一組密碼保護（比照 mala-staff-eval 的通行碼做法），密碼存 Script Properties，不寫在前端。

5. 五條主要流程

5.1 建合約單

你在 admin 填欄位 → 選房間後床位選單才出現、月租金自動帶入 → 租期起日填了自動算迄日（+6 個月 −1 天）→ 床位若已有「在住」合約，跳警告但允許強制建立（交接期重疊）→ 產生 https://eason0728.github.io/mala-dorm/sign.html?t=32碼，你自己複製傳給同仁。

5.2 同仁簽合約

開連結 → 封面 → 十八條＋附件一（含賠償單價表）＋附件二規約，全部唯讀，只有設備點收可勾 → 手寫簽名（canvas，可清除重畫；未簽名不能送出）→ 送出 → 後端產 PDF → 畫面顯示「已完成」並提供下載按鈕（第十六條「各自下載留存」的實作）。

同一條連結再開一次：顯示唯讀的已簽版本＋下載按鈕，不能再送出。

5.3 產生 PDF

以一份 Google Docs 範本為底，Apps Script 複製範本 → 取代 {{姓名}} {{房間}} {{租期}} 等佔位符 → 插入手寫簽名圖與大章圖 → 匯出 PDF → 存進雲端硬碟 → 刪掉暫存 Doc。

技術風險（未確認，Phase 0 先驗）：Apps Script 另一條路是 HTML 直接轉 PDF， 寫起來比較快，但中文字型在 Google 的轉檔器上有掉字風險。 所以主方案選 Google Docs 範本（中文穩、封面分頁好控制）。 動工第一件事就是花半小時做一個最小驗證：產一份含中文＋簽名圖＋大章的 PDF， 確認字沒破、圖有在、封面自己一頁。過了才往下寫。

5.4 自動續約（第二條）

每天 03:00 的時間觸發器掃 status=在住 的合約：

  • 距 term_end 剩 30 天 → 後台清單標示，並寄一封 Email 給 madesiaosinla@gmail.com（Eason 2026-07-23 追加）。 當天沒有到期在即的合約就不寄；同一筆只寄一次（notified_at 欄防重複）； 多筆合併成一封。
  • 過了 term_end 且你沒標記「已通知終止」→ term_start/term_end 各推六個月、term_no +1、 寫一列 audit_log。不重簽、不重產 PDF（條文寫的就是「不另行簽訂契約」）。
  • 你若在期滿前標記「已通知終止／換房」→ 到期日轉為 待退宿，可直接開點交單。

5.5 退宿點交

你在後台對該筆按「開點交單」→ 產生第二條連結 → 同仁逐項勾選正常／異常、歸還／未還 → 勾「需清潔」時加 3,000 → 應賠償金額即時加總顯示 → 手寫簽名送出 → 產點交 PDF（出租人欄位蓋大章）→ 合約 status 轉「已退宿」，床位釋出。

6. 安全與存證

  • token：Utilities.getUuid() 去掉連字號再串一次，32 字元，猜中機率可忽略。一人一單一碼，簽完即失效。
  • 存證三件組：簽署時間（伺服器時間，不信任前端）、IP、User-Agent，寫進 contracts 與 audit_log 兩處。
  • 簽名圖：PNG 存雲端硬碟，同時嵌進 PDF，兩份互為佐證。
  • 條文版本：每份合約記 terms_ver，日後改條文不影響既簽合約重印時的內容。
  • 個資最小化：只存身分證字號，不上傳影本；試算表與資料夾不開共用連結，只有你的帳號能看。
  • 保存年限：第十八條第四項寫五年，實作上就是不刪雲端硬碟資料夾（不做自動刪除）。

9. 驗收條件

  1. Phase 0 的 PDF 驗證：中文不破字、簽名圖與大章都在、封面獨立一頁。
  2. 建一筆合約單拿到連結；床位重複時會跳警告。
  3. 手機開連結：先看到封面，往下是十八條＋兩份附件，iPhone 上不需橫向捲動。
  4. 未簽名送不出去；簽名可清除重畫。
  5. 送出後雲端硬碟出現 YYYY-MM-DD_姓名_宿舍合約.pdf，內容與畫面一致。
  6. 同一連結再開＝唯讀＋可下載，不能重送。
  7. 試算表三張表都有對應紀錄，audit_log 有時間／IP／UA。
  8. 把某筆的 term_end 改成昨天、手動跑一次續約函式 → term_no +1、日期推六個月、audit_log 有一列。
  9. 點交單七項勾選＋需清潔 → 應賠償金額計算正確，PDF 上的數字與畫面一致。
  10. 點交送出後合約狀態轉「已退宿」，該床位可以建新合約且不再跳重複警告。
